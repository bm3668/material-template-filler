# 材料模板填充 Web 应用 - 产品介绍

> 📋 智能填充各类项目申报/竞赛材料模板，让文档撰写效率提升 10 倍

---

## 📖 产品概述

**材料模板填充 Web 应用**是一款基于 AI 的智能文档填充工具，专为项目申报、竞赛材料、基金申请等场景设计。用户上传模板和项目说明后，系统自动识别模板结构，智能提取项目内容，一键生成填充完整的申报材料。

### 核心价值

- ⚡ **效率提升**：从数小时手动填写 → 1 分钟自动填充
- 🎯 **精准匹配**：AI 智能识别模板字段，自动匹配对应内容
- 📊 **质量可控**：智能字数控制，填充报告透明可查
- 🔒 **隐私安全**：本地化部署，数据不出服务器

## ✨ 核心功能

### 1. 智能模板解析

- 🔍 自动识别 Word 模板结构（标题样式、表格字段）
- 📋 提取字段填写要求（字数限制、内容要点）
- 🏷️ 智能字段分类（个人信息/项目内容）

### 2. 多源内容输入

支持三种项目内容提供方式：

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 📝 **直接输入** | 在网页对话框中输入项目信息 | 快速填写，内容简洁 |
| 📎 **上传文件** | 上传项目说明文档（.md/.txt/.docx） | 已有详细项目文档 |
| 📂 **历史复用** | 选择之前上传的项目说明 | 同一项目申报多个模板 |

### 3. 智能内容填充

- 🤖 AI 智能匹配：将项目内容映射到模板字段
- 📏 字数控制：自动扩展/精简内容，符合字数要求
- 🔄 别名处理：识别相似字段名，避免重复填充
- ⚠️ 置信度标注：标注每个字段的填充来源和可信度

### 4. 填充报告生成

每份填充结果附带详细报告：

```
📊 填充摘要
├── ✅ 已基于材料内容填充：5 个字段
├── ⚠️ LLM 扩展生成内容：10 个字段
└── ❌ 未填充：2 个字段

📝 各模块填充详情
├── 摘要：✅ 70% 置信度，300 字
├── 设计目标：✅ 70% 置信度，500 字
└── ...
```

### 5. Web 交互界面

三步式操作流程：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  ① 上传模板  │ →  │  ② 提供内容  │ →  │  ③ 下载结果  │
│   .docx     │    │  输入/上传   │    │  .docx+.md  │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    Web 前端                              │

│  - 结果下载                                              │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              Flask 后端服务                              │
│  - 文件管理（上传/存储/下载）                            │
│  - 任务调度（填充任务队列）                              │
│  - API 接口（RESTful）                                   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│        material-template-filler 核心引擎                 │

│  ├── content_matcher.py    内容匹配引擎                 │
```

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | HTML5 + CSS3 + JavaScript | 响应式设计，支持拖拽上传 |
| **后端** | Python 3 + Flask | 轻量级 Web 框架 |
| **文档处理** | python-docx | Word 文档解析与生成 |
| **内容匹配** | 规则引擎 + 关键词匹配 | 智能字段识别与内容提取 |
| **内容扩展** | LLM（可选） | 智能内容生成与扩展 |

## 📖 使用流程

### 步骤 1：上传模板

```
1. 点击"上传模板"区域
2. 选择或拖拽 .docx 模板文件
3. 系统自动解析模板结构
```

**支持模板类型**：
- 大学生创新项目申报书
- 互联网 + 竞赛商业计划书
- 挑战杯竞赛作品申报书
- 国家自然科学基金申请书
- 各类政府资助项目申报书

### 步骤 2：提供项目内容

**方式 A：直接输入**
```
项目名称：AI 辅助学习系统
项目背景：当前在线教育平台缺乏个性化推荐...
研究目标：开发智能学习推荐系统...
团队成员：3 人
项目周期：6 个月
预算：2 万元
```

**方式 B：上传文件**
- 上传已有的项目说明文档
- 支持 .md、.txt、.docx 格式
- 系统自动提取内容

**方式 C：历史复用**
- 从列表中选择之前上传的项目说明
- 同一项目可填充多个模板

### 步骤 3：执行填充

```
1. 点击"🚀 开始填充"
2. 系统自动处理（通常 1-3 分钟）
3. 实时查看处理日志
```

### 步骤 4：下载结果

- 📥 **下载填充后的文档** - 完整的 .docx 文件
- 📊 **查看填充报告** - 详细的填充状态说明

### 快速安装


## 📝 典型应用场景

### 场景 1：大学生创新项目申报

**背景**：某高校学生团队准备申报国家级大学生创新项目

**痛点**：
- 申报书模板复杂，包含 20+ 字段
- 团队首次申报，缺乏经验
- 截止日期临近，时间紧迫

**使用流程**：
1. 上传《国家级大学生创新项目申报书》模板
2. 上传团队准备的项目计划书（.docx）
3. 点击"开始填充"
4. 下载填充后的申报书和填充报告
5. 根据报告检查需要人工补充的内容

**效果**：
- ⏱️ 填写时间：3 小时 → 10 分钟
- 📝 人工补充：仅需完善团队信息和个人签名
- ✅ 申报成功率：提升（格式规范、内容完整）

### 场景 2：互联网 + 竞赛多模板处理

**背景**：参赛团队需要同时准备校赛、省赛、国赛材料

**痛点**：
- 三个级别模板相似但有差异
- 需要反复调整内容适配不同模板
- 容易遗漏或填错信息

**使用流程**：
1. 第一次：上传校赛模板 + 项目说明 → 填充
2. 第二次：上传省赛模板 + 选择历史项目说明 → 填充
3. 第三次：上传国赛模板 + 选择历史项目说明 → 填充

**效果**：
- 🔄 内容复用：一次输入，多次使用
- 📋 一致性：确保核心信息一致
- ⚡ 效率：3 个模板总耗时 <30 分钟

### 场景 3：科研基金批量申报

**背景**：高校科研处需要协助多位老师申报各类基金

**痛点**：
- 多位老师，多个项目，多个模板
- 格式要求各异，容易出错
- 重复性工作多

**使用流程**：
1. 收集所有老师的 projects 说明文档
2. 批量上传到系统（inputs/目录）
3. 逐个模板进行填充
4. 分发给对应老师检查确认

**效果**：
- 📊 批量处理：支持多项目并行
- 👥 分工明确：科研处负责格式，老师负责内容
- 📈 申报质量：格式规范性显著提升
