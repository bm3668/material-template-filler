# 材料模板填写 Skill

智能填充各类项目申报/竞赛材料模板。

## 触发场景

- "帮我把这个项目内容填到模板里"
- "根据我的描述填写申报材料"
- "/fill-template [模板路径] with [项目内容]"
- "用 templates/申报书.docx 帮我填"

## 目录结构

```
~/.openclaw/workspace/
├── templates/              # 模板文件目录
│   └── *.docx             # 各类申报/竞赛模板
├── inputs/                 # 项目说明文档目录
│   └── *.md/*.docx/*.txt  # 用户上传的项目说明（自动存档）
└── filled/                 # 输出目录
    └── *_filled_*.docx    # 填充后的文档
    └── *_report_*.md      # 填充报告
```

## 如何提供文件

### 📄 提供模板文件

**方法 1：指定本地路径（推荐）**
```
用户：/fill-template /home/admin/.openclaw/workspace/templates/申报书模板.docx
用户：项目内容是：我有一个大学生创新项目...
```

**方法 2：工作区相对路径**
```
用户：用 templates/基金申请书.docx 帮我填
用户：项目内容是：...
```

**方法 3：直接上传**
在对话中直接上传 .docx 文件，然后说"请帮我填这个模板"

### 📝 提供项目说明文档

**方法 1：直接输入（自动存档）**
```
用户：/fill-template templates/申报书.docx
用户：
项目名称：AI 辅助学习系统
项目背景：当前在线教育...
研究目标：开发智能推荐...
```
→ 自动保存到 `inputs/项目名_YYYYMMDD_HHMMSS.md`

**方法 2：上传文档文件**
上传 `.md` / `.docx` / `.txt` 文件后说：
```
用户：用这个文档填充 templates/申报书.docx
```
→ 自动保存到 `inputs/文件名_YYYYMMDD_HHMMSS.原扩展名`

**方法 3：使用历史项目说明**
```
用户：列出历史项目说明
用户：用 inputs/xxx.md 填充 templates/xxx.docx
```

## 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 接收输入                                                  │
│     - 直接文本 → 自动存档到 inputs/                          │
│     - 上传文件 → 自动存档到 inputs/                          │
│     - 历史文件 → 直接读取                                     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 解析模板                                                  │
│     - 识别标题模块（Heading 样式）                            │
│     - 识别表格字段                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 理解项目说明                                              │
│     - 提取关键信息（名称、背景、目标、团队、预算等）           │
│     - 结构化存储                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 智能匹配                                                  │
│     - 将内容映射到模板模块                                    │
│     - 计算置信度评分                                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. 填充文档                                                  │
│     - 生成填充后的 .docx 文件 → filled/                       │
│     - 生成填充报告 .md → filled/                             │
└─────────────────────────────────────────────────────────────┘
```

## 支持的模板类型

- ✅ **标题样式模板**：使用 Heading 样式的结构化文档
- ✅ **表格字段模板**：包含表单表格的竞赛/申报模板
- ✅ **混合模板**：同时包含标题和表格的文档

## 输出

### 📤 填充结果

所有生成的文件都保存在 `workspace/filled/` 目录下：

- **填充完成的 .docx 文件**（`filled/原文件名_filled_YYYYMMDD_HHMMSS.docx`）
- **填充报告 .md 文件**（`filled/原文件名_fill_report_YYYYMMDD_HHMMSS.md`）

### 📥 项目说明存档

用户提供的项目说明自动保存到 `workspace/inputs/` 目录：

- **直接输入** → `inputs/项目名_YYYYMMDD_HHMMSS.md`
- **上传文件** → `inputs/原文件名_YYYYMMDD_HHMMSS.扩展名`

### 📋 报告内容

报告详细说明每个模块的填充状态：

### 报告内容

报告详细说明每个模块的填充状态：

| 状态 | 标识 | 说明 |
|------|------|------|
| ✅ 已基于材料内容填充 | 高置信度 (≥70%) | 从用户提供的材料中直接提取并填充 |
| ⚠️ LLM 扩展生成内容 | 中置信度 (30%-70%) | 内容不足，由 AI 智能扩展生成，建议检查 |
| ❌ 未填充 | 低置信度 (<30%) | 未找到相关内容，需人工补充 |

报告包含：
- 整体填充率统计
- 各模块详细填充情况（状态、置信度、字数、内容预览）
- 需要人工补充的模块列表
- 建议检查的模块列表

## 注意事项

### 文件格式
- ✅ 模板：仅支持 `.docx` 格式（不支持旧版 `.doc`）
- ✅ 项目说明：支持 `.md` / `.docx` / `.txt` 格式

### 文件管理
- 模板中的标题样式会被识别为模块
- 填充后的文件会添加"_filled"后缀
- 项目说明自动存档到 `inputs/`，可重复使用
- 建议填充后人工检查确认

### 最佳实践
1. **首次使用**：准备详细的项目说明文档（建议 Markdown 格式）
2. **多次填充**：使用历史存档的项目说明，避免重复输入
3. **批量处理**：一个项目说明可填充多个相关模板
4. **检查报告**：查看填充报告，重点关注"⚠️ LLM 扩展"和"❌ 未填充"的模块
