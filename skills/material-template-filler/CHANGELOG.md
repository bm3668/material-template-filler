# Changelog - 材料模板填充 Skill

## [2.0.0] - 2026-03-21

### ✨ 重大更新：DocxFiller V2 引擎

基于 **Anthropic docx skill** 最佳实践重构的专业级文档填充引擎。

#### 新增功能

- **📐 页面设置标准化**
  - 支持 A4/A3/Letter/Legal 等多种纸张尺寸
  - 支持纵向/横向布局切换
  - 标准边距预设（normal/narrow/wide）
  - 自动检测模板纸张尺寸

- **🎨 专业样式管理系统**
  - 统一覆盖 Heading 1-9 样式
  - 正文字体、字号、行距标准化
  - 中英文字体映射（黑体/Arial Black, 宋体/Arial）
  - 跨平台字体兼容性（Windows/macOS/Linux）

- **📊 表格样式优化**
  - 自动应用 Table Grid 样式
  - 智能识别字段类型（个人信息 vs 项目内容）
  - 支持字数限制自动截断
  - 保持单元格格式一致性

- **✓ 文档验证系统**
  - ZIP 结构完整性检查
  - XML 格式正确性验证
  - 必需文件存在性检查
  - 生成后自动验证

- **📋 增强填充报告**
  - 结构化报告数据（JSON 格式）
  - 模块级填充状态追踪
  - 置信度评分可视化
  - 缺失个人信息清单

#### 改进点

| 功能 | V1 | V2 | 改进 |
|------|-----|-----|------|
| 页面设置 | 依赖模板 | 标准化 A4/边距 | ✅ 一致性提升 |
| 样式管理 | 无 | 专业样式系统 | ✅ 质量提升 |
| 字体兼容 | 英文字体 | 中英文映射 | ✅ 跨平台支持 |
| 文档验证 | ❌ 无 | ✅ ZIP/XML 验证 | ✅ 可靠性提升 |
| 填充报告 | 简单文本 | 结构化数据 | ✅ 可编程分析 |
| 占位符匹配 | 基础 | 模糊匹配增强 | ✅ 准确率提升 |

#### API 变更

```python
# V1 用法（仍然兼容）
filler = DocxFiller(template_path, output_path)
filler.fill(content_map)

# V2 推荐用法
filler = DocxFillerV2(template_path, output_path)
report_data = filler.fill(content_map)  # 返回结构化报告
filler.add_fill_report()  # 添加详细报告到文档
```

#### 性能对比

基于测试模板（7 个模块，4 个表格字段）：

| 指标 | V1 | V2 | 变化 |
|------|-----|-----|------|
| 填充时间 | 0.04 秒 | 0.10 秒 | +150% |
| 文件大小 | 37.5 KB | 38.2 KB | +2% |
| 文档验证 | ❌ 不支持 | ✅ 通过 | - |
| 填充准确率 | 85% | 89% | +4% |

V2 增加了样式管理和文档验证，性能略有下降但显著提升了文档质量和可靠性。

## 版本说明

### 版本号规则

采用语义化版本号：`MAJOR.MINOR.PATCH`

- **MAJOR** - 不兼容的 API 变更（如 V2 引擎）
- **MINOR** - 向后兼容的功能新增
- **PATCH** - 向后兼容的问题修复

### 升级建议

- **V1 用户** - 建议升级到 V2，获得更好的文档质量和可靠性
- **新用户** - 直接使用 V2 引擎

### 兼容性

- V2 与 V1 API 向后兼容
- 模板文件格式不变
- 输出文件格式不变
